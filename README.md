# 任务要求

基于区块链的众筹系统

- 参考petshop实验，通过web3和ganache、Hardhat 等工具完成前端界面、合约编写、区块链部署、
  功能展示。

- 前端不做过多要求，可使用vue、react等前端框架完成。

-  功能要求

  - 项目创建：允许用户提交项目名称、描述、筹款目标金额和截止日期 等信息。合约需要存储
    这些项目信息，并为每个项目分配一个唯一的 ID。

  - 资金捐赠：允许用户选择一个正在进行中的项目，并向其捐赠一定数量的币，存入合约中。
    合约需要记录每个捐赠者的地址和捐赠金额，并更新项目的当前筹款总额。能够展示所有正
    在进行和已结束的众筹项目的信息，包括名称、描述、目标金额、已筹金额、剩余时间和捐
    赠者列表。

  - 筹款结束逻辑：基于时间结束，当达到项目的截止日期时，任何人都可以调用一个合约函数
    来结束该项目。结束结果判断：

    - 1. 达到或超过目标： 合约将当前筹集到的所有资金转移给项目发起人。

    - 2. 未达到目标： 合约允许捐赠者提取他们捐赠的资金。
      即：项目发起人在项目成功结束后，可以调用一个合约函数将筹集到的资金提取到他的账
      户。捐赠者在项目失败后，可以调用一个合约函数将他们捐赠的资金提取回来。

  - 可选形式：网页、小程序等。

- 可选额外内容（额外加分）

  - 实现基于目标完成度的阶段性资金释放 (简化版里程碑)，捐款达到一定比例后，或者在项目
    发起人标记完成某个里程碑后 ，允许释放一部分资金。（比如达到了某个阈值后可以先提取
    一部分钱出来）
  - 增加基于时间的早期捐赠奖励机制，例如，前 10 位捐款人可以获得额外的感谢或未来可能的
    福利。

# 基于区块链的众筹系统

这是一个参考 pet-shop Hardhat 实验结构完成的众筹 dApp。合约负责项目创建、捐赠、结束结算、成功提现、失败退款，并包含两个加分功能：基于筹款比例或发起人标记的阶段性资金释放、前 10 位早鸟捐赠者排名与奖励记录。

## 功能

- 创建项目：提交项目名称、描述、目标金额和截止日期，每个项目自动获得唯一 ID。
- 资金捐赠：用户选择进行中的项目捐赠 ETH，合约记录捐赠者地址和金额。
- 项目展示：前端展示进行中和已结束项目的名称、描述、目标金额、已筹金额、剩余时间、捐赠者和早鸟捐赠者。
- 截止结算：截止后任何人都可以调用结束函数。
- 成功提现：达到或超过目标后，项目发起人可以提现。
- 失败退款：未达到目标时，捐赠者可以取回自己的捐款。
- 加分功能：筹款达目标 50% 或发起人标记里程碑完成后，可提前释放 30% 阶段性资金；前 10 位捐赠者会被记录为早鸟支持者并显示排名。

## 运行

安装依赖：

```shell
npm install
```

运行测试：

```shell
npm test
```

启动本地区块链：

```shell
npm run node
```

新开终端部署合约到本地链：

```shell
npm run deploy:localhost
```

启动前端：

```shell
npm run dev
```

浏览器打开 `npm run dev` 输出的本地地址，使用 MetaMask 连接 `http://127.0.0.1:8545` 本地网络，并导入 Hardhat 控制台输出的测试账户私钥即可演示。
